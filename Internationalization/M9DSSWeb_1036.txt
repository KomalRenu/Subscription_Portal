Un périphérique est un appareil spécifique pour fournir un contenu de souscription à l'utilisateur final.  Chaque adresse créée dans le portail par un utilisateur est conçue pour n'utiliser qu'un périphérique, comme Outlook 98 ou 2000.
Le type de périphérique est un groupe permettant à l'utilisateur de choisir le moyen par lequel il recevra le contenu. Lorsqu'une nouvelle définition de site est créée, quatre types de périphériques standard seront créés et configurés automatiquement.
Les types sont :
Les assistants numériques personnels
Les téléphones portables
Les récepteurs d'appels
Le courrier électronique
Ces périphériques par défaut sont créés lorsque le Narrowcast Portal Designer de MicroStrategy est installé et que les dossiers associés à ces périphériques sont stockés dans la base de registre Narrowcast.
Cet assistant met les options suivantes à votre disposition :
renommer un type de périphérique défini
modifier une définition de type de périphérique
supprimer un type de périphérique devant être pris en charge par le portail
éditer le dossier de type de périphérique
créer un nouveau type de périphérique
Pour chaque nouvelle définition de type de périphérique, vous devrez spécifier le nom du type de périphérique, l'icône de l'URL étendue et réduite, le format de l'adresse et les champs qui seront utilisés lors de l'affichage et l'édition.
Ce champ vous permet d'ajouter ou de supprimer des éléments à partir du dossier pour ce type de périphérique défini.
Les éléments à gauche sont disponibles, mais ne sont plus actifs sur ce type de périphérique.  Les éléments à droite sont en cours de sélection pour ce type de périphérique spécifique.
Dans ce contexte, un périphérique est une façon définie de fournir un contenu à l'utilisateur final.  Les paramètres par défaut sont "client messagerie électronique générique" et "périphérique de portail".  Vous pouvez modifier ces paramètres par défaut.
Cliquez sur le périphérique approprié pour modifier la sélection.
Pour chaque source d'informations, vous devrez définir si les utilisateurs doivent entrer leurs informations d'authentification, comme un nom d'utilisateur et un mot de passe.
Si vous utilisez les références du projet, le mécanisme d'authentification sera ignoré et les utilisateurs seront transférés vers les informations qu'ils ont demandées.
Vous pouvez aussi exiger les références de l'utilisateur.
La valeur par défaut est de ne pas permettre aux utilisateurs d'entrer d'authentification pour des sources d'informations.
Vous devrez modifier l'exigence si vous souhaitez mettre des mesures d'authentification en place.
